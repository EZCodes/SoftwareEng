Description of the graph

This graph compares Microsoft and Google Open Source involvement, as well as compares their engagements in those projects
The sizes are proportional to amount of lines committed to the respective repositories.
  • Amount of repositories
  • Amount of lines on all repositories
  • Lines of code committed by employees and lines of code committed by non-employees of respective company
  • Lines committed in specific languages by either employees or non employees
These metrics allow us to see a lot of things. First we can see how involved in open source are these two companies in comparison to each other
We can also check how engaged companies are. If a company have more non-employees committing the their repositories, that means that they are not engaged and just leaving it to people from outside.
We can also see if there's difference between languages of employees or non-employees.
It might be the case that employees will commit in languages that this company developed, while non-emlpoyees will commit in something else
We are able to check, what are the main languages of the company repositories, and compare that to above metrics

Summarizing

This graph allows us to compare engagement in open source between Microsoft and Google and the tools that they are use or allow to use.

Analysis of Data

As we can see on the graph, Microsoft has way more Open Source Projects than Google. However, both companies tend to leave the projects in the hands of non-employees
It does make sense, since they probably have more important internal projects to do instead. In terms of languages it seems like that for Microsoft projects the most common are
  • C++
  • C
  • A bit of Java and JavaScript
  • Combination of other, less common languages
Its good to note that Employees and Non-Employees use similar languages, with exception of JavaScript, which is not popular amongst employees

In case of Google, the languages are:
  • JavaScript
  • Python
  • C
  • HTML
  • Bit of Java
  • Large presence of other, less common languages
It also should be noted, that while these are most used languages, Employees only tend to use Java, while the rest of the languages is committed mostly by Non-Employees
